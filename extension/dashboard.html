✏️ Naming Conventions

Define how folders and files are named when stories are saved to disk. Use the tokens below — if a token's data is not available for a story it is removed cleanly, with no dangling dashes or empty segments. Illegal Windows characters and emojis are stripped automatically. File names are truncated to a maximum of 100 characters. Cross-use: [StoryDate] / [StoryTitle] / [CentreName] / [EducatorName] work here too (same values as [Date], [Title], [Daycare]/[Centre], [Educator]).

Controls how story folders are structured inside your download directory. Slash characters (/) create sub-folders. If a token is not available for a story, it is removed automatically along with its surrounding delimiter.
[ChildName] [Daycare] [Class] [Date] [Title] [StoryDate] [StoryTitle] [CentreName] [EducatorName] [OriginalName] [Child] [Room] [Centre] [Educator]
Controls how individual media files (photos and videos) are named. The original file extension (.jpg, .mp4, etc.) is always preserved. Works for all media types.
[ChildName] [Daycare] [Class] [Date] [Title] [StoryDate] [StoryTitle] [CentreName] [EducatorName] [OriginalName] [Child] [Room] [Centre] [Educator]
